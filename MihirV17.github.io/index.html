# Medieval Mayhem!! ## A familiar game with a fun twist #### Inspired by Google’s offline dinosaur game, medieval mayhem presents a classic favorite with a medieval twist. Move the dragon up and down to dodge castles and arrows, all while watching your score go higher and higher! ![Alt text](images/Dragon.PNG) ## Game Features ![Alt text](images/Backgroun1.PNG) ### COLLISION DETECTION #### We’ve implemented collision detection in our game, so the player won’t be able to take on our challenge easily. The images have bounding boxes that have been mostly fitted to the objects, which provides the player with an easy way of avoiding obstacles. ### Randomized Obstacle Speeds #### Randomized obstacle speeds have been added so the player has more of a challenge. Arrows come flying out of nowhere and will really test the reaction speed of the player. Maneuverability is key, and we hope it provides a more challenging experience! ### Switching Image Animation #### As an appealing final visual touch, the dragon switches between two image states whenever the player presses a movement key. The dragon looks like it is flying, soaring from the user’s imagination. ### Updating Score #### Our score feauture constantly updates as the player moves through the state of the game. It a great way to keep track of how your game is going. ### Start, Exit & Try Again Buttons #### We included ease-of-access buttons so it’s really simple to navigate through our game. Our start button on the start screen, exit and try again buttons on the end screen are intuitive, allowing you to exit. ![Alt text](images/End_screen.PNG) ![Alt text](images/start_screen.PNG) ## Installation #### Our program is open source and available on GitHub. You can install the program using the link below. The program can be played using the play_game.py file (shown to the right). Have fun playing our game! ![Alt text](images/Code.PNG) ## Program Elements #### The program contains files that run Medieval Mayhem. #### startscreen.py: initializes our starting screen #### player.py: initializes our player object #### background.py: initializes the background #### obstacle.py: initializes arrows and obstacle objects #### view.py: draws the objects onto the game world #### controller.py: initializes a controller object #### model.py: saves the state of each object per iteration #### endscreen.py: initializes our end_screen #### test_obstacle.py: obstacle pytests #### test_model.py: pytests for the model ## About ### Kenta Terasaki ##### ![Alt text](images/Kenta.jpg) ### Prisha Bhatia ##### ![Alt text](images/Prisha.jpg) ### Mihir Vemuri ##### ![Alt text](images/Mihir.jpg)
